# RAPG Candidate-Selector Surrogate Preflight

## 结论边界

当前实验不是完整 autoregressive RAPG Pilot 0。它在 V4.4 的固定 `{A,B}` 候选对上，由冻结 LoRA 的 mean-sequence-logprob softmax 定义并实际采样一个 candidate-selector policy，用于低成本检验 gradient-aware auditing 是否值得进入 clean on-policy 数据重采。

即使 gate 通过，也只授权：**采集 clean on-policy autoregressive RAPG bank**。不授权 policy training 或论文正向算法声明。

## 数据与成本

- 从 raw `candidate_events.jsonl` 导出全部 replay-valid、both-valid pair；不得按 Rescue/Reverse/zero delta 筛选。
- 物理拆分 public events、executed B return、private Shadow-A return。
- cohort 仍以 replay validity 为条件，因此是 development stress set，不代表部署流。
- cross-fit allocator 消耗的历史 Full-Shadow source labels 单独报告，不计入“20%在线 audit”是假设性 conditional pretraining cost；因此不能声称端到端同总成本。

## 主指标

主指标不使用低维梯度投影。对固定 committed propensity，精确重算完整 LoRA score-norm design MSE：

```text
MSE_design = N^-2 sum_i ||s_i||^2 (1-p_i)/p_i (Y_i-m_i)^2.
```

投影 gradient MSE 和 cosine 仅为 secondary diagnostics。

## 方法与 gate

- Uniform、Residual-only、Score-only、RAPG `w*sigma`、Oracle ceiling。
- fixed `c=1`，20% expected audits，`p_min=0.05`，primary 不裁剪 `1/p`。
- 至少 100 events；RAPG design MSE 比 Uniform 低至少 15%，并优于 Residual-only。
- task improvement fraction ≥50%，max positive task share ≤50%，ESS ≥5，max weight ≤20。
- Gate 从 source/bank/predictions/behavior ledger/propensity ledger/audit draws 独立重算，不信任 summary 常量。
- 服务器 Torch 核心测试不得 skip。
